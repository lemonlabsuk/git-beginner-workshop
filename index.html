How Git Works

Why Git Works This Way?

  • Allows you to commit your work-in-progress often
  • Work effectively without connectivity
  • Fast - no network lag

Signup on Github

  1. Register an account at github.com
  2. Note: Do not go into the settings and turn on Two-factor authentication. It makes things more complicated in the workshop
  3. Send Ian Forsey your username

Install Github Desktop

  1. Go to desktop.github.com
  2. Download and install!
  3. Run it!
  4. Sign in with your github credentials
  5. Click Finish

Git Clone - it's like svn checkout

Get a new copy of a repo on your machine

  1. Go to this Github repository
  2. Click the button
  3. Copy the repo URL wiht the button
  4. In Github Desktop navigate to File > Clone a Repository
  5. Click the URL tab
  6. Paste the Repo URL, pick a local path and click Clone!

Git Commit - it's like svn commit

Commits a change, but doesn't send it to the server!

  1. Make a new file firstname.surname.txt, using *your* name e.g. joe.bloggs.txt
  2. Make the contents of the file I love Git!
  3. In Github Desktop, make sure it is checked in the Changes tab
  4. Enter a summary, description and click Commit to master

Note: New untracked files need to be added with git add (just the same as svn add). Github Desktop handles this for us.

Git Log - it's like svn log

Check the commit history

  1. In Github Desktop, click the History tab
  2. You should see your commit!
  3. In Github, click the button
  4. You should not see your commit on the server!

Git Push - it's a little bit like svn commit

Push your local commits to the server

  1. In Github Desktop, click the Repository > Push
  2. What happens?

Oh no!

  • Cannot Push, try pulling first / Non fastforward updates were rejected
  • This is like the working copy is out of date error in SVN

Git Pull - it's like svn update

Pull does a fetch and a merge

  1. In Github Desktop, click Repository > Pull and Repository > Push again
  2. You'll need to do this a few times, if there are many people!
  3. In Github, click the button
  4. You should now see your commit! and everyone elses' in the room!

Git Branch - it's like svn copy

Creates a branch in your local repo

  1. In Github Desktop, click Branch > New Branch
  2. Name the branch firstname-surname, using *your* name e.g. joe-bloggs
  3. Add the line I love branches! to the file you created earlier
  4. Click Repository > Push. This will push your branch+commit to the server!
  5. In Github, click the button, and you should see your branch
